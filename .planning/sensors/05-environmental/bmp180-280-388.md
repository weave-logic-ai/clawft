## Overview

Bosch's BMP family is barometric pressure + temperature (no humidity). In WeftOS this file focuses on the *environmental* angle — weather sensing, altitude, floor detection. A separate entry in [category 02](../02-positioning-navigation/) or [03](../02-positioning-navigation/) may cover the altimetry/IMU fusion use case; cross-link as appropriate.

- **BMP180** — legacy; 5 V-tolerant; slow; limited resolution.
- **BMP280** — cheap workhorse; same footprint as BME280; I²C + SPI.
- **BMP388 / BMP390** — high-resolution (~0.03 hPa / ~25 cm altitude), lower noise, FIFO, preferred for modern builds.

## Key specs

| | BMP180 | BMP280 | BMP388 |
|-|--------|--------|--------|
| Range | 300–1100 hPa | 300–1100 hPa | 300–1250 hPa |
| Rel. accuracy | ± 1.0 hPa | ± 1.0 hPa | ± 0.08 hPa (↑ resolution) |
| Abs. accuracy | ± 1.5 hPa | ± 1.0 hPa | ± 0.5 hPa |
| Altitude resolution | ~20 cm | ~17 cm | ~8 cm |
| Interface | I²C (0x77) | I²C (0x76/77) / SPI | I²C (0x76/77) / SPI |

## Interface & wiring

- BMP280/BMP388: same I²C pinout as BME280. SDO selects address.
- BMP180: I²C only, 5 V tolerant on many breakouts; deprecated for new builds.
- Pressure measurement is quiet electrically but sensitive to wind / HVAC drafts — keep the sensor in a baffled enclosure for weather use.

## Benefits

- Very low power — suitable for battery weather nodes.
- BMP388 gives floor-change altitude resolution (you can detect someone walking up stairs).
- Simple I²C / SPI interface; no gas burn-in like BME680.
- Can provide T compensation for other sensors if you trust its T channel (±1 °C).

## Limitations / gotchas

- No humidity — if you need T/RH/P in one chip, use [BME280](./bme280.md) instead.
- Absolute altitude requires a known reference pressure; "altitude from barometer" drifts as weather changes by tens of meters per day.
- BMP180 is obsolete; don't design in. (Also shares 0x77 with BME280, which matters on legacy gear.)
- BMP388 FIFO fill interrupts are useful but easy to misconfigure — read Bosch app note if using them.

## Typical use cases

- Weather station barometer trend.
- Floor / height change detection in multi-story buildings.
- Drone / rocket altimetry (paired with IMU in [category 03](../02-positioning-navigation/)).
- Tight pressure tracking for HVAC / cleanroom monitoring.
