## Overview

The BME280 is Bosch's workhorse combined T/RH/P sensor: ±1 °C, ±3 % RH, ±1 hPa, 1 Hz or faster, tiny package, I²C or SPI, and well-supported drivers everywhere. For WeftOS it's the default T/RH/P sensor for indoor nodes and weather stations.

## Key specs

- Temperature: −40 to +85 °C, ± 1.0 °C.
- Humidity: 0–100 % RH, ± 3 %.
- Pressure: 300–1100 hPa, ± 1 hPa (~1 m altitude resolution).
- Interfaces: I²C (addresses 0x76 or 0x77 via SDO pin) and SPI.
- Power: very low average current; deep sleep between samples.

## Interface & wiring

- I²C: `SDA`, `SCL`, plus `VCC` (1.71–3.6 V, most modules are 3.3 V only), `GND`. SDO selects 0x76 (GND) or 0x77 (VCC) — matches default of BMP280, so watch for clashes.
- SPI: `MOSI`, `MISO`, `SCLK`, `CS`; up to 10 MHz.
- CSB pin on the chip must be tied high for I²C mode (on most breakouts that's already done).
- Keep the sensor away from PCB self-heat (USB regulators, CPU) or RH will read low by 5–10 %.

## Benefits

- Excellent accuracy for the price; tiny footprint.
- I²C + SPI flexibility; well-supported in every language.
- Low current — fine on coin-cell nodes with duty-cycling.
- Altitude via pressure → a free "floor / height" reading in buildings.

## Limitations / gotchas

- **RH is self-heating sensitive.** If the sensor PCB gets warm from its host MCU, humidity reads low. Mount remote or on a thermal-break daughterboard.
- BME280 vs BMP280 share the same footprint and I²C addresses; cheap clones sometimes ship BMP280 (no humidity) labeled as BME280 — read chip ID (0x60 for BME280, 0x58 for BMP280) at boot.
- 0x76/0x77 often collides with other sensors; use a [TCA9548A mux](../08-communication/tca9548a-i2c-mux.md) if needed.
- RH accuracy drifts ~1 % / year; recalibrate against a known reference annually.

## Typical use cases

- Indoor environment telemetry.
- Weather station baseline (T/RH/P).
- Altitude / floor detection in multi-story buildings.
- Grow tent / greenhouse.

## Where to buy

- Adafruit / SparkFun (verified genuine Bosch parts, calibrated).
- AliExpress — cheapest, but 30–40 % of "BME280" listings are actually BMP280.

## Software / libraries

- `bme280` crate (Rust) is pure `embedded-hal`, works on ESP32 and RP2040.
- Adafruit / SparkFun Arduino libraries are the reference implementations.
- ESP-IDF component `esp-idf-lib` for C.

## Notes for WeftOS

- At boot, read the chip ID register and refuse to advertise "humidity" capability if the chip is a BMP280.
- Apply a simple Kalman filter on pressure if deriving altitude, to smooth weather-front noise.
- Store "last calibrated against reference" timestamp; UI should warn after 1 year.
