## Overview

ST7789 (and its older sibling ST7735) are Sitronix TFT controllers found on small IPS displays: 1.14" 135×240, 1.3" 240×240, 1.54" 240×240, 1.8" 128×160, 2.0" 240×320. They share a driver family with [ILI9341](./ili9341-tft.md) via the `mipidsi` crate, so WeftOS can treat them as interchangeable surfaces. The 1.3" 240×240 square module is especially common on ESP32 "smart watch" dev boards.

## Key specs

- ST7789: resolutions up to 240×320, 16/18-bit color, IPS panels (wide viewing angle).
- ST7735 (older): up to 128×160, lower contrast TN panels on cheap modules.
- Interface: 4-wire SPI up to 60+ MHz.
- Voltage: 3.3 V logic; on-board LDO on most breakouts.
- Rarely ships with a touch panel (unlike ILI9341); usually just a display.

## Interface & wiring

- `SCLK`, `MOSI`, `CS`, `DC`, `RES`, plus `LED` backlight.
- Some square 240×240 ST7789 boards omit the `CS` line — the chip is hard-enabled. You can't share SPI with another peripheral in that case; add a FET or get a CS-broken-out version.
- Short wires above 40 MHz SPI; ESP32 DMA works well up to 60 MHz.

## Benefits

- IPS = excellent viewing angle and color accuracy for the price.
- Small form factors fit wearables and tight enclosures.
- Same driver stack as ILI9341 — code is portable between sizes.
- No dedicated touch controller means simpler BOM when you only need display.

## Limitations / gotchas

- **No-CS boards** on the 240×240 square — check before designing a shared SPI bus.
- Offset / mirrored rendering is common: small modules often have a 40 px offset on one axis; must set via the `set_offset()` / `orientation` call.
- ST7735 is lower resolution and sometimes has washed-out colors (TN not IPS) — pick ST7789 unless you specifically want 1.8" 128×160.
- Backlight is typically not PWM-isolated; drive via MOSFET for silent dimming.

## Typical use cases

- Wrist-worn or badge-style UIs (240×240 square).
- Compact telemetry screen on a desk node.
- Secondary color display alongside an e-paper main.

## Software / libraries

- `mipidsi` crate (Rust) — supports both ST7789 and ST7735 with the same API, via `embedded-graphics`.
- `TFT_eSPI` (Arduino / PlatformIO) is still the fastest option with DMA.

## Notes for WeftOS

- Provide a single HAL surface type `TftSpi` with configurable resolution/offset/orientation; ST7789 and ILI9341 both implement it.
- Register the no-CS quirk in device config so the SPI scheduler knows not to attempt bus sharing.
- Round corner masking for badge UIs should live at the compositor layer, not in each app.
