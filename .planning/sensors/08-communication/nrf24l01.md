## Overview

The nRF24L01+ is Nordic's 2.4 GHz ShockBurst transceiver — the cheap, weird,
proprietary radio that predated BLE. It runs a packet-based protocol with
hardware auto-ACK, retransmits, and six "pipes" (logical addresses) per node.
Two nRF24L01+ modules talk to each other out of the box, and a "mesh-ish"
star can be built with one primary and up to six peers per pipe group. It
does **not** interoperate with BLE; this is its own protocol.

The long-range `+PA+LNA` variant with a screw-on antenna pushes the range to
hundreds of meters outdoors for another dollar.

## Key specs

| Spec | Value |
|------|-------|
| Band | 2.4 GHz ISM (125 channels, 1 MHz apart) |
| Data rate | 250 kbps / 1 Mbps / 2 Mbps |
| TX power | −18 to 0 dBm (20 dBm on +PA+LNA variant) |
| Pipes | 6 per node |
| Supply | 1.9 – 3.6 V — **not 5 V tolerant on VCC** |
| Current | ~12 mA TX, ~14 mA RX, ~900 nA sleep |

## Interface & wiring

Eight pins: `VCC / GND / CE / CSN / SCK / MOSI / MISO / IRQ`. Share SPI with
other peripherals, dedicate `CSN` and `CE`. **Do not power from 5 V** —
you'll see magic smoke. On the other hand, the digital inputs tolerate 5 V
just fine, so an older 5 V Arduino can drive the SPI lines directly.

Add a **10 µF electrolytic across VCC/GND right at the module** — the #1
"why doesn't my nRF24 work" is insufficient decoupling on the 3.3 V rail.

## Benefits

- Cheap as dirt — $1 for bare modules.
- Hardware auto-ACK + retransmit = reliable packets without host effort.
- ~900 nA sleep is excellent for battery nodes.
- `+PA+LNA` variant genuinely reaches 500 m+ outdoors.

## Limitations / gotchas

- **Decoupling cap.** Without a 10 µF cap the TX current spikes collapse
  the 3.3 V rail; the module looks broken but is starved.
- **Fake / counterfeit parts** on AliExpress are common; some only do 1
  Mbps despite claiming 250 kbps support.
- Proprietary protocol; won't talk to BLE, WiFi, or anything off-brand (aside
  from similar Beken BK2423 chips).
- Not encrypted. Anyone with another nRF24 can sniff — build your own crypto
  on top if it matters.
- Range in-building is modest. Don't assume +PA+LNA outdoor numbers apply
  through drywall.

## Typical use cases

- Wireless RC / drone links.
- Cheap home-automation point-to-point sensors when ESP-NOW would work but
  you want lower power.
- Multi-node farm sensors that don't need a WiFi AP.
- Keyboards / controllers where BLE is overkill.

## Software / libraries

- `RF24` (nRF24/RF24) — canonical Arduino library.
- `RadioLib` also supports nRF24 with the same API as LoRa / CC1101.
- ESPHome does not support nRF24 natively; write a custom component.

## Notes for WeftOS

Tag nRF24 links as `encryption=none` unless you layer AES on the payload;
WeftOS rules must not treat nRF24 traffic as authenticated by default.
