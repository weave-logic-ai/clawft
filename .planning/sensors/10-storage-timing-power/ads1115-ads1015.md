## Overview

The ADS1115 (16-bit) and ADS1015 (12-bit) are TI delta-sigma ADCs over I²C.
Four single-ended or two differential input channels, internal programmable
gain amplifier (PGA) with ±0.256 V to ±6.144 V ranges, and an internal
reference — so readings are stable regardless of supply noise. They're the
canonical escape hatch from the ESP32's infamously poor built-in ADC.

## Key specs

| Spec | ADS1115 | ADS1015 |
|------|---------|---------|
| Resolution | 16-bit | 12-bit |
| Channels | 4 SE / 2 diff | 4 SE / 2 diff |
| Sample rate | 8 – 860 sps | 128 – 3300 sps |
| Input range (PGA) | ±0.256 V … ±6.144 V | same |
| Internal reference | yes, factory-trimmed | yes |
| I²C address | 0x48 – 0x4B (4 variants) | same |

## Interface & wiring

- I²C at 0x48 (solder `ADDR` to move to 0x49 / 0x4A / 0x4B — four per bus).
- Supply 3.3 V from the ESP32.
- `ALRT/RDY` pin signals conversion complete or threshold compare — wire to a
  GPIO if you're doing continuous high-rate sampling.
- Differential mode (e.g., AIN0–AIN1) rejects common-mode noise — use for
  bridge sensors and long cable runs.

## Why external ADC? (ESP32 ADC gotchas)

This is the **main reason** to reach for an ADS1115. The ESP32's internal ADC
has all of the following problems:

- **Non-linearity.** The SAR ADC reads are non-monotonic and off by tens of mV
  at the rails. Espressif recommends a calibration routine using fused
  reference voltages, which is mandatory, not optional.
- **Wi-Fi cross-talk.** The radio couples 5–50 mV of noise into the ADC
  reference while transmitting — you can literally see Wi-Fi activity on the
  analog channel.
- **ADC2 is blocked while Wi-Fi is active.** ADC2's channels (which include
  GPIO 4, 12, 13, 14, 15, 25, 26, 27) return garbage or fail entirely any time
  the radio is up. Many dev boards expose ADC2 pins as "just ADC" with no
  warning.
- **Dynamic range** is effectively 10–11 bits after noise averaging — the
  nominal "12-bit" figure is marketing.

An ADS1115 gives you a noise floor of < 100 µV, true 16-bit resolution, and
complete isolation from the radio's switching noise. For any project measuring
small signals (load cells, gas sensors, temperature bridges), use an external
ADC.

## Benefits

- 16-bit resolution, internal reference — actually accurate.
- Four channels per chip, up to four chips per bus (16 channels total).
- Programmable gain covers a huge range in one part.
- Conversion-complete interrupt on `ALRT/RDY` for clean sampling.

## Limitations / gotchas

- ADS1115 is **slow**: 860 sps max. For higher rates use ADS1015 (3.3 ksps) or
  SAR ADCs like the MCP3208.
- I²C bus becomes the bottleneck above ~500 sps — short traces, 400 kHz bus.
- Input pins can be driven no more than 0.3 V beyond rails — add series
  resistors or a clamp for user-exposed inputs.

## Typical use cases

- Precision reads of pots / joysticks / strain gauges.
- MQ / CO₂ / soil-moisture analog sensors (see
  [`../05-environmental/mq-series.md`](../05-environmental/mq-series.md)).
- Load cells via [`../11-wild-card/hx711-load-cell.md`](../11-wild-card/hx711-load-cell.md)
  — but HX711 is purpose-built; ADS1115 is for general-purpose analog.
- Battery voltage monitoring via a resistor divider.

## Notes for WeftOS

The substrate's **default analog source** should be ADS1115 when any precision
is needed. WeftOS should surface "ADC source = ESP32-internal" as a capability
warning in node metadata because it implies all the gotchas above.
