## Overview

The MAX30102 is an integrated **pulse-oximetry + heart-rate** front end. On a single
die it combines a red LED (~660 nm), an IR LED (~880 nm), a photodiode, analog
front-end, 18-bit ADC, and an I²C digital interface. You shine red + IR into a
fingertip or earlobe, measure the light that makes it back, and the ratio of red
to IR absorption gives you SpO₂; the pulsatile component gives you heart rate (PPG).

The **MAX30105** is a pin-compatible upgrade with a third green LED, useful for
particle / proximity / smoke experiments as well as PPG.

## Key specs

| Spec | Value |
|------|-------|
| LEDs | Red + IR (30102); Red + IR + Green (30105) |
| ADC | 18-bit, programmable sample rate 50 Hz–3.2 kHz |
| Bus | I²C, up to 400 kHz fast-mode |
| Supply | 1.8 V logic, 3.3 V LEDs (modules accept 3.3V / 5V) |
| Current | ~600 µA active, < 1 µA shutdown |
| FIFO | 32-sample on-chip FIFO |

## Interface & wiring

Most hobby breakouts expose `VCC / GND / SCL / SDA / INT`. Wire SCL / SDA to any
ESP32 I²C pair; the `INT` pin lets the chip wake the MCU when the FIFO fills.
The finger must be **held still and warm** against the glass window; cold fingers
and movement wreck SNR.

## Benefits

- All-in-one: no external LEDs, amps, or analog design required.
- FIFO and interrupts → low-power duty-cycled sampling.
- Well-supported open-source libraries with HR + SpO₂ algorithms.

## Limitations / gotchas

- Consumer-grade only, **not medical**. Readings are easily fooled by motion, poor
  contact, skin tone variation, and ambient IR.
- SpO₂ algorithms in most open libraries are reference implementations, not
  clinically validated. Do not ship health claims.
- The IR LED is bright enough that loose modules can appear "on" through clothing;
  power-gate in sleep scenarios.

## Typical use cases

- Wearable HR band with OLED readout.
- Desk-side "stress / focus" indicator fed into WeftOS activity graph.
- Fitness project telemetry, or finger-clip prototype for learning PPG.

## Software / libraries

- SparkFun's `MAX3010x` library is the de-facto Arduino / ESP-IDF driver and
  includes HR and SpO₂ example sketches.
- ESPHome exposes the chip via its `max30102` sensor platform for Home Assistant.

## Notes for WeftOS

PPG readings should enter WeftOS as a **biometric-scoped** stream and must not be
written to shared memory namespaces without user opt-in. When used only as a
"person is touching the device" proxy, the derived boolean is fine to share; the
raw waveform is not.
