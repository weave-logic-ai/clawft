# Sketches — NOT FINAL

These are **rough directional sketches**, not design specs. They exist
to anchor conversation about *one* possible vibe, not to dictate it.

## `01-beach-crab-dock.jpg`
- Full-bleed wallpaper (beach + crab-with-glasses mascot).
- A faint warped-grid overlay sits atop the photo (the dark-matter
  displacement language from the current egui shell).
- Bottom **dock** — frosted-glass capsule, rounded corners, 8 glyph-only
  tiles. Spacing generous, icons monochromatic light-on-dark with a
  subtle blue-white bias.
- Top-right status row (time, wifi, battery-like glyphs).

## `02-cosmos-crab-dock.jpg`
- Starfield + galaxy wallpaper, crab perched on a rock (same mascot).
- Same dock structure, this time with **named apps**: `Weft`,
  `Navigator`, `Supervisor`, `Cosmos`, `ExoChain`, `Mesh`, `Forge`,
  `Control`.
- Icons have color (Weft blue "W", Cosmos purple spiral, Mesh blue
  node-dot-grid, Forge/anvil motif, etc.).

## What to take from them
- A named starter app set is worth proposing (see RQ5 / RQ7).
- The dock as a universal surface (launcher + live status chips).
- Wallpaper is context (photo / render / procedural) — the surface
  draws over it, the warped grid is one possible overlay language.
- Mascot presence is deliberate — the system has personality.

## What **not** to take
- The heavy glass/blur is not automatically the answer. "Crisp Linux"
  language may push us toward flatter, less-material surfaces. The
  symposium decides.
- The full-colour icons clash with the stated "no-brand-color
  identity at the base." Probably fine as a later identity pass,
  not as the primitive set.
- macOS-ness is a reference, not a target. We want our own thing.
