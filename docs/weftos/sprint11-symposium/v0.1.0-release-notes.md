# WeftOS v0.1.0 Release Notes

**Release Date**: 2026-03-28
**Tag**: `v0.1.0`

---

## What Is WeftOS

WeftOS is a portable AI kernel written in Rust. It provides process management, inter-process communication, a cognitive substrate, mesh networking, and governance enforcement for AI agents. It runs on Linux, macOS, Windows, and WebAssembly. It is designed to understand, document, and automate client systems through a knowledge graph.

This is the foundation release. It establishes the kernel (K0-K6), the cognitive loop (DEMOCRITUS), the operator CLI (Weaver), and the framework CLI (weft). The 22-crate workspace compiles to three binaries that can be deployed as a single node, a multi-node mesh, or embedded in a browser.

---

## Highlights

- **Complete kernel in 6 layers**: K0 boot, K1 supervisor with RBAC, K2 IPC with pub/sub and delegation, K3 WASM tool sandbox, K4 container lifecycle, K5 application framework, K6 mesh networking and agent agency.
- **DEMOCRITUS cognitive loop**: Continuous sense-embed-search-update-commit cycle that builds a causal model of the codebase. Processes git events, file changes, and IPC messages into a vector-indexed knowledge graph.
- **Self-healing supervisor**: Erlang-inspired restart strategies (OneForOne, OneForAll, RestForOne, Escalate) with budget tracking and exponential backoff. Processes link and monitor each other.
- **Mesh networking**: Bind, accept, handshake, dispatch across nodes. mDNS discovery. A2A bridge between local kernel messages and mesh IPC envelopes. Two-node LAN communication demonstrated.
- **Constitutional governance**: 22 genesis rules anchored to an ExoChain. Three-branch model (executive, legislative, judicial). All cross-process mutations require chain-logged approval.
- **5,040 tests across 22 crates**: 1,596 kernel tests alone. 20/20 gate checks passing. Full-stack integration tests covering native agent, WASM sandbox, container lifecycle, and ECC substrate.
- **Three distribution binaries**: `weft` (framework CLI), `weaver` (operator/daemon CLI), `weftos` (standalone AI kernel binary).

## What's Included

### Added

#### Kernel Foundation (K0-K6)

- WeftOS kernel crate (`clawft-kernel`) with 7 layered phases
- K0: Boot sequence, process table, capability registry, interactive REPL console
- K1: AgentSupervisor with RBAC capability enforcement, agent lifecycle, ExoChain integration
- K2: A2A IPC with topic pub/sub, inbox routing, request-response messaging, delegation wiring
- K2.1: SpawnBackend, ServiceEntry, GovernanceGate; chain-logged lifecycle signals
- K2-ext: RVF deep integration with signing, witness chains, governance hooks, lineage tracking
- K3: WASM tool sandbox with Wasmtime execution, fuel metering, tool registry integration
- K3c: ECC cognitive substrate with 6 kernel modules (causal_graph, hnsw, impulse_queue, embedding, cognitive_tick, cross_ref)
- K4: Container types, config, lifecycle manager, ServiceApi, dual governance gate, chain contracts
- K5: Application framework with manifest parsing (`weftapp.toml`), CLI commands, lifecycle management, namespace isolation
- K6a: Cluster, environment, and governance types for mesh networking
- K6b: Agent-first architecture with roles, agency model, autonomous operation

- Self-healing supervisor: RestartStrategy enum, restart budget, exponential backoff, ProcessLink, ProcessMonitor
- Persistence: SQLite backend for ExoChain, CausalGraph save/load, HNSW index persistence

- MeshRuntime: bind/accept/handshake/dispatch, A2A bridge, mDNS discovery
- DEMOCRITUS continuous cognitive loop: Sense, Embed, Search, Update, Commit with budget-aware pacing
- ConfigService and AuthService as kernel services
- Tool signing with Ed25519 for tamper detection
- WASM shell execution: ShellCommand/ShellResult types, execute_shell() in Wasmtime sandbox

#### Weaver Operator CLI

- `weaver` binary as WeftOS operator CLI
- Persistent kernel daemon with Unix socket RPC
- Kernel event log with `weaver kernel logs`
- Signal-based stop/restart for systemd compatibility

- Codebase analysis pipeline: 12 conversation sources ingested

#### Framework CLI (`weft`)

## Installation

### From Source (Recommended for v0.1.0)

```bash
# Requires Rust 1.93+
git clone https://github.com/clawft/clawft.git
cd clawft
scripts/build.sh native
```

The binaries are placed in `target/release/`:
- `weft` -- framework CLI for agent sessions, channels, tools
- `weaver` -- operator CLI for kernel management, daemon control
- `weftos` -- standalone AI kernel binary

### Docker

```bash
docker pull ghcr.io/clawft/clawft:0.1.0

# Run the gateway
docker run -p 8080:8080 ghcr.io/clawft/clawft:0.1.0

# Run with persistent state
docker run -v weftos-data:/home/weft/.clawft ghcr.io/clawft/clawft:0.1.0
```

### Binary Download

Pre-built binaries for Linux (x86_64, aarch64), macOS (x86_64, Apple Silicon), and Windows (x86_64) are attached to the [GitHub Release](https://github.com/clawft/clawft/releases/tag/v0.1.0).

### Shell Installer

```bash
# Unix/macOS
curl -fsSL https://github.com/clawft/clawft/releases/latest/download/clawft-cli-installer.sh | sh

# Windows PowerShell
irm https://github.com/clawft/clawft/releases/latest/download/clawft-cli-installer.ps1 | iex
```

## Known Limitations

- **No crates.io publish**: The crates are not yet published. Install from source or binary download.
- **Ruvector path dependencies**: The `exo-resource-tree` crate requires a sibling `../ruvector/` checkout for the `rvf-crypto` dependency. This will be feature-gated in v0.1.1.
- **WASM target mismatch**: `scripts/build.sh` targets `wasm32-wasip1` while CI targets `wasm32-wasip2`. Will be unified in v0.2.0.
- **Dockerfile builds `weft` only**: The `weaver` and `weftos` binaries are not yet included in the Docker image. They can be built from source.
- **K8 GUI prototype**: The Tauri desktop app is a prototype. It requires separate Tauri build tooling and is not included in the release binaries.
- **ONNX embeddings require model download**: The ONNX embedding backend needs a sentence-transformer model file downloaded separately. No bundled model is included.
- **No `cargo audit` or `cargo deny` in CI**: Supply-chain security checks are planned for v0.1.1.
